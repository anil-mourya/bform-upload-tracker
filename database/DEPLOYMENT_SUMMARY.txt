DATABASE SCHEMA SPECIFICATION
================================================================================

TABLES: 4

1. bform_societies (56 records)
   Columns:
   • id (UUID, PRIMARY KEY)
   • customer_id (VARCHAR 50, UNIQUE) - B-Form ID (168BF, etc.)
   • society_name (VARCHAR 255)
   • batch_type (ENUM: batch_1, batch_2, batch_3)
   • status (VARCHAR 20) - active, inactive, suspended
   • contact_email (VARCHAR 100)
   • contact_phone (VARCHAR 20)
   • last_upload_date (TIMESTAMP)
   • created_at, updated_at (TIMESTAMP)

2. bform_uploads (main tracking table)
   Columns:
   • id (UUID, PRIMARY KEY)
   • customer_id (VARCHAR 50, FK to societies)
   • batch_type (ENUM)
   • year (INTEGER)
   • period (VARCHAR 20)
   • file_name, file_path (VARCHAR, TEXT)
   • file_size (BIGINT)
   • upload_date (TIMESTAMP)
   • status (ENUM: pending, processing, completed, rejected, archived)
   • uploaded_by (VARCHAR 100)
   • rejection_reason (TEXT, nullable)
   • number_of_forms (INTEGER)
   • checksum (VARCHAR 64, for duplicate detection)
   • created_at, updated_at, deleted_at (TIMESTAMP)

   Constraints:
   • UNIQUE (customer_id, year, period, deleted_at)
   • CHECK year >= 2000 AND year <= 2100
   • CHECK file_size >= 0
   • CHECK (status != 'pending' OR rejection_reason IS NULL)

3. bform_upload_history (audit trail)
   Columns:
   • id (UUID, PRIMARY KEY)
   • upload_id (UUID, FK with CASCADE)
   • previous_status (ENUM, nullable)
   • new_status (ENUM)
   • changed_by (VARCHAR 100)
   • changed_at (TIMESTAMP)
   • change_reason (TEXT)
   • ip_address (VARCHAR 45)
   • created_at (TIMESTAMP)

4. bform_upload_summary (statistics)
   Columns:
   • id (UUID, PRIMARY KEY)
   • year (INTEGER)
   • period (VARCHAR 20)
   • total_uploads (INTEGER)
   • completed_uploads (INTEGER)
   • pending_uploads (INTEGER)
   • processing_uploads (INTEGER)
   • rejected_uploads (INTEGER)
   • archived_uploads (INTEGER)
   • total_file_size (BIGINT)
   • total_forms (INTEGER)
   • batch_completion_rate (NUMERIC 5,2)
   • last_upload_date (TIMESTAMP)
   • last_updated_at, created_at, updated_at (TIMESTAMP)

   Constraints:
   • UNIQUE (year, period)
   • CHECK year >= 2000 AND year <= 2100

INDEXES: 12

Single-Column Indexes (8):
1. idx_bform_uploads_customer_id ON bform_uploads(customer_id)
2. idx_bform_uploads_year ON bform_uploads(year)
3. idx_bform_uploads_period ON bform_uploads(period)
4. idx_bform_uploads_upload_date ON bform_uploads(upload_date)
5. idx_bform_uploads_status ON bform_uploads(status)
6. idx_bform_uploads_batch_type ON bform_uploads(batch_type)
7. idx_bform_upload_history_upload_id ON bform_upload_history(upload_id)
8. idx_bform_upload_history_changed_at ON bform_upload_history(changed_at)

Composite Indexes (4):
1. idx_bform_uploads_year_period ON bform_uploads(year, period)
2. idx_bform_uploads_customer_year_period ON bform_uploads(customer_id, year, period)
3. idx_bform_upload_summary_year_period ON bform_upload_summary(year, period)
4. Additional supporting indexes

Note: All indexes include soft-delete filter (WHERE deleted_at IS NULL)

ENUMS: 2

1. upload_status
   ├─ pending (awaiting processing)
   ├─ processing (currently processing)
   ├─ completed (successfully processed)
   ├─ rejected (processing failed)
   └─ archived (archived/inactive)

2. batch_type
   ├─ batch_1 (33 societies: 168BF-200BF)
   ├─ batch_2 (7 societies: 232BF-238BF)
   └─ batch_3 (16 societies: 250BF-265BF)

TRIGGERS: 8

Data Calculations (1):
