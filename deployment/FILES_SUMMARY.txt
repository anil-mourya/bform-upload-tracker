QUICK REFERENCE GUIDE
================================================================================

FOR LOCAL DEVELOPMENT:
1. Copy backend.env.example -> backend/.env
2. Copy frontend.env.example -> frontend/.env
3. Run: docker-compose up -d
4. Access: http://localhost:3000

FOR RAILWAY DEPLOYMENT:
1. Run: chmod +x setup-railway.sh && ./setup-railway.sh
2. Follow interactive prompts
3. Deployment automatically handled
4. Monitor: railway logs --follow

FOR DATABASE MANAGEMENT:
- Backup:  ./backup-database.sh production
- Restore: ./restore-database.sh backups/file.sql.gz production
- Migrate: railway shell backend --command "npm run migrate"

FOR CI/CD:
1. Copy railway-deploy.yml to .github/workflows/
2. Add GitHub secrets: RAILWAY_TOKEN, RAILWAY_PROJECT_ID, etc.
3. Push to main/staging to trigger automatic deployment
